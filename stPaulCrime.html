St. Paul Crime Map

This is a project where I worked on a team of 3, and we had to create an interactive, responsive, single-page application. Our page would use the Leaflet and Nominatim and RESTful API to allow the user investigate crimes from verious neighborhoods within St. Paul, MN. The Vue.js framework was also used to assist in making the application to the user interactions. My group received top marks for the assignment.

Following the instructions within the README on github, one can also load up this application as a local host. We provided a number of interactions, including: an address that will automatically shift the map to that location, crime types to filter only certain types of crimes at a time, neighborhoods for specific areas, and other filters like time.

The table at the bottom and the map will update as the filters at the top are changed. On top of this, we also have the crimes update as the user changes where they are looking on the map. For example, when zooming in on a specific location, the table will update so that only crimes that would be visible on the map will show up on the table. Finally, we added the ability to add the crime to the map so that the user can see exactly where the crime took place.
